## Overview

`@unifast/plugin-sanitize` enables HTML sanitization in the unifast pipeline to protect against XSS (cross-site scripting) attacks. It strips or filters dangerous HTML tags, attributes, and URL protocols from the compiled output. Sanitization is enabled by default when the plugin is applied.

Use this plugin when processing untrusted Markdown/MDX input (e.g., user-generated content).

## Type Definitions

### `SanitizePluginOptions`

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `enabled` | `boolean` | `true` | Enable or disable sanitization |
| `schema` | `SanitizeSchema` | - | Custom sanitization schema |

### `SanitizeSchema`

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `allowedTags` | `string[]` | - | HTML tag names to allow (all others are stripped) |
| `allowedAttributes` | `Record<string, string[]>` | - | Map of tag name to allowed attribute names |
| `allowedProtocols` | `Record<string, string[]>` | - | Map of attribute name to allowed URL protocols |

### Produced `CompileOptions`

```ts
{
  sanitize: {
    enabled: boolean,
    schema?: SanitizeSchema,
  }
}
```
