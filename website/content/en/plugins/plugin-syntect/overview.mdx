## Overview

`@unifast/plugin-syntect` enables syntax highlighting for fenced code blocks using [syntect](https://github.com/trishume/syntect), a Rust-native syntax highlighting library powered by Sublime Text syntax definitions. This runs entirely inside the Rust compilation pipeline with zero JavaScript overhead and supports 100+ languages out of the box.

For JavaScript-based highlighting with VS Code themes, see [`@unifast/plugin-shiki`](/docs/plugins/plugin-shiki/overview).

## Type Definitions

### `SyntectPluginOptions`

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `engine` | `"none" \| "syntect"` | `"syntect"` | `"syntect"` uses the syntect highlighter; `"none"` disables highlighting |

### Produced `CompileOptions`

```ts
{
  highlight: {
    enabled: true,
    engine: "none" | "syntect",
  }
}
```

## Comparison with `@unifast/plugin-shiki`

| Feature | `plugin-syntect` | `plugin-shiki` |
|---------|-----------------|----------------|
| Runtime | Rust (native) | JavaScript (Shiki) |
| Performance | Fastest | Slower (async init) |
| Output | CSS classes (`sy-*`) | Inline styles or CSS |
| Languages | 100+ (Sublime syntaxes) | 200+ via Shiki |
| Themes | CSS-based | Full VS Code theme support |
| Setup | Synchronous | Async (`await createShikiPlugin()`) |

Use `plugin-syntect` for maximum performance with broad language coverage. Use `plugin-shiki` when you need VS Code themes or very niche language support.
